# ステップ1: iPhone版 要件整理・移植計画

作成日: 2026-07-17

## 1. 目的

Android版を変更せず、ヤクピタの主要業務をiPhoneで実行できる独立アプリを新規開発する。

この文書では開発範囲、技術方針、段階的な進め方を定義する。ステップ1ではSwiftコードやXcodeプロジェクトをまだ作成しない。

各技術試作は使い捨てにせず、品質条件を満たした実装とテストを本アプリへ継続利用する。試作で問題が判明した技術や構造だけを、レビューと判断記録を経て差し替える。

## 2. 絶対条件

- Android版のプログラム、設定、既存ドキュメント、画像、音声を編集しない。
- iPhone版の変更は `ios-yakupita/` 内だけで行う。
- 医薬品照合では、読取不能を一致として扱わない。
- カメラ権限拒否、データ未登録、マスター更新失敗から安全に復帰できる設計にする。
- 各開発ステップの終了時にレビューを実施し、指摘を解消してから次へ進む。

## 3. 対象機能

Android版の現行仕様から、次をiPhone版の候補範囲とする。

1. 初回施設登録、利用者選択・登録
2. MEDIS医薬品マスターの取得・保存・更新
3. 調剤モード
   - JAHIS QR読取
   - 処方内容の解析と期待薬リスト表示
   - PTP・箱のGTIN読取と照合
   - 合計量表示
4. 帳票監査モード
   - 帳票撮影と日本語OCR
   - 読取結果の確認
   - PTP・箱のGTIN照合
5. 充填モード
   - 充填元と充填先のバーコード照合
   - 履歴表示
6. 医薬品検索、利用者登録マスター、プライバシー表示
7. 成功・警告の音、振動フィードバック

機能の厳密な同等性は各ステップで受入条件として具体化する。

## 4. 技術方針

- UI: SwiftUI
- 状態管理: Swift ConcurrencyとObservable系API
- カメラ・バーコード: AVFoundationを基本とし、必要に応じてVisionを併用
- 日本語OCR: Vision
- 永続化: SwiftDataまたはSQLite。データ量と検索性能を試作で比較して確定
- ネットワーク: URLSession
- テスト: XCTest。JAHIS解析、GTIN正規化、数量計算、照合判定を優先する

Android固有のCameraX、Room、ML Kit、Jetpack ComposeをiOS側へ直接持ち込まない。

## 5. 段階計画

### ステップ1: 要件整理・移植計画

本書を作成し、Android版と分離した作業境界を確定する。

### ステップ2: Mac・Apple開発環境の確認

Mac、Xcode、対象iPhone、Apple ID、Bundle ID候補を確認し、対応iOSバージョンを決める。

### ステップ3: 独立Xcodeプロジェクトの作成

SwiftUIアプリ、単体テスト、UIテストのターゲットを作り、起動確認する。

### ステップ4: カメラ読取の技術試作

カメラ権限、GTINバーコード読取、結果表示、連続読取抑止、エラー復帰を実機確認する。検証済みコードとテストは本アプリのカメラ基盤として残し、後続機能から利用する。

### ステップ5: 共通業務ロジック

GTIN正規化、照合判定、JAHIS解析、数量計算をSwiftで実装し、Android版の既存テストケース相当を再現する。

### ステップ6: マスターと基本設定

施設・利用者・医薬品マスターの保存、検索、更新、失敗時の復旧を実装する。

### ステップ7: 3モードの順次実装

小さく検証できる単位に分け、充填、調剤、帳票監査の順を基本候補として進める。優先順位は着手前に利用目的と実機試験のしやすさから確定する。

### ステップ8: 品質・配布準備

アクセシビリティ、個人情報、プライバシー文書、クラッシュ対策、TestFlight、App Store申請資料を整備する。

## 6. ステップ1の完了条件

- iPhone版の作業領域がAndroid版から分離されている。
- 対象機能と非対象技術が明記されている。
- 次の作業が開発環境確認であると明記されている。
- Android版ファイルに差分がない。
- `codex-review` の指摘が解消されている。
