# 利用者登録マスター

## 目的

帳票監査モードで、MEDISの公開マスターに含まれない院内製剤や医療材料を
薬品候補として選択し、ピッキング時にバーコード照合できるようにする。

## 操作

1. ホーム画面右上のメニューを開く。
2. `院内製剤・材料マスター` を選ぶ。
3. 帳票に記載される名称、ピッキング時に読むバーコード、必要に応じて規格・包装と区分を入力する。
4. `マスターに登録` を押す。

GTIN-13 / GTIN-14 は従来どおりGTIN-14へ正規化して保存する。
院内製剤や材料の施設独自バーコードは、3〜64文字の読み取り値を照合キーとして保存する。

## データ設計

- 利用者登録品は既存の `drug_master` に `isUserRegistered = true` で保存する。
- `yjCode` と `drugCode` には `USER-<正規化済みバーコード>` を設定する。
- このコードにより帳票OCRの候補とピッキング対象が同じ同一性を共有する。
- MEDIS手動取込・自動更新では `isUserRegistered = false` の公開マスターのみ削除する。
- 利用者登録品は登録画面から個別削除できる。

## 復旧・再構築時の注意

Roomデータベースはバージョン11で、10からの移行時に
`drug_master.isUserRegistered INTEGER NOT NULL DEFAULT 0` を追加する。
公開マスター更新処理で `drug_master` を全件削除すると利用者登録品が失われるため、
必ず `DrugMasterDao.deleteImported()` を使う。

## 確認項目

- 登録した名称が帳票監査の手動検索候補に出る。
- 登録したバーコードを帳票PTPスキャン画面で読むと該当行が確認済みになる。
- MEDIS更新後も利用者登録品が残る。
- 登録画面で削除した品目は候補とバーコード照合から消える。
